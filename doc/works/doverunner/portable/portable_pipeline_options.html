Portable 배포 — Bitbucket Pipeline 옵션 상세

custom 파이프라인 manual-build-portable의 입력 변수 9개 레퍼런스

실행 방법

Portable 이미지는 push로 자동 실행되지 않고 수동 전용 custom 파이프라인으로만 빌드됩니다.

  1. Bitbucket 리포지토리 → Pipelines → Run pipeline
  2. 빌드할 브랜치 선택
  3. Pipeline에서 Custom: manual-build-portable 선택
  4. 아래 9개 변수 폼 입력 (기본값 미리 채워짐) → Run
  5. 완료 후 실행 화면 Artifacts 탭에서 portable/** 다운로드
ℹ
값 검증은 Bitbucket이 아니라 빌드 중 ci/build_portable.sh가 수행합니다(형식·필수 조건 불일치 시 빌드 실패). validate_env.sh는 이 파이프라인에 포함되지 않습니다.

배포 예제 (owl-app → owl-ssm)

Portable 배포는 두 리포지토리의 파이프라인을 순서대로 실행합니다. ① owl-app이 SecureOwl을 빌드해 환경별 S3에 배포 → ② owl-ssm이 그 결과를 받아 portable 이미지(.tar)를 빌드합니다.

① owl-app 파이프라인 → S3: <ENV>/<VERSION>/secure_owl + manifest → ② owl-ssm manual-build-portable → portable 이미지 .tar
⚠
두 단계의 ENV(환경)와 VERSION은 반드시 일치해야 합니다 — owl-ssm은 owl-app이 그 환경·버전으로 S3에 올린 SecureOwl을 내려받기 때문입니다.

* 필수* = 기본값이 채워져 있어 비우지만 않으면 됨(파이프라인이 기본값 제공).

⚠ 꼭 확인할 주의사항

  • API — none만 완전 오프라인. 그 외(dev/tst/svc)는 런타임에 외부 API 연결. 또한 CI는 API 값과 무관하게 항상 스냅샷(실 자격증명 포함)을 임베드합니다.
  • ENV ≠ API — ENV는 "SecureOwl 출처", API는 "런타임 연결 대상". 서로 독립이니 혼동 금지.
  • EXPIRE_DATE — UTC 자정 기준 만료, 지나면 컨테이너 기동 실패. 로컬 스크립트 기본은 +1일로 매우 짧음 → 길게 쓰려면 명시 입력.
  • REPORT=custom이면 REPORT_URL 필수 — 비우면 빌드 실패.
  • 검증 주체 — Bitbucket이 아니라 빌드 중 build_portable.sh가 검증. VERSION 등 자유 입력값은 형식 검증이 없으니 정확히 입력.
  • S3 리포지토리 변수(SECURE_OWL_S3_BUCKET 등)는 폼이 아니라 Bitbucket 설정에 미리 등록돼 있어야 함.

옵션 상세

ENV — SecureOwl 소스 환경

기본값 dev선택지 dev / tst / svc드롭다운
설명빌드에 넣을 SecureOwl 리소스를 어느 환경의 S3에서 받아올지 결정합니다.
영향download_secure_owl.sh가 s3://<bucket>/<ENV>/<VERSION>/secure_owl/ 와 .../manifest.json 경로를 구성. build-info.json에도 기록.
⚠주의 — API와 혼동 금지: ENV는 "빌드 재료(SecureOwl) 출처", API는 "런타임에 붙을 API". 서로 독립입니다.

API — 런타임 API 환경 주의

기본값 dev선택지 dev / tst / svc / none드롭다운
설명완성된 이미지가 실링 시 어느 API에 연결할지. none이면 외부 연결 없이 완전 오프라인으로 동작(내장 오프라인 API 서버 사용).
영향build_portable.sh가 ^(dev|tst|svc|none)$ 검증 후 config.yaml의 api: 라인에 기록 → 런타임 API_ENV로 주입. none이면 api_base_url 미로딩(placeholder).
⚠주의: 로컬 빌드 스크립트는 API=none일 때만 스냅샷 캡처+WBC 암호화를 수행하지만, CI는 API 값과 무관하게 항상 스냅샷을 캡처·임베드합니다(개선 대상으로 식별된 동작 — was_properties의 자격증명 포함).

VERSION — 버전

기본값 3.0.0.0자유 입력
설명서비스/빌드 버전 (예: 3.0.0.0).
영향S3 경로 세그먼트, snapshot service_version 파라미터, 이미지 태그 prefix, 라이선스/sealing_option 버전, Go ldflags에 모두 사용.
⚠주의: Bitbucket·이 파이프라인에서 형식 검증은 하지 않습니다(자유 문자열). 정확한 값을 입력하세요.

EXPIRE_DATE — 라이선스 만료일 주의

기본값 20270513형식 YYYYMMDD자유 입력
설명Portable 라이선스 만료일. 비워두면 빌드일 기준 +1년으로 자동 계산.
영향build_portable.sh가 ^[0-9]{8}$ 검증 후 → 내장 lc의 expire(Unix 타임스탬프, UTC 자정) + license.json의 expireDate(YYYYMMDD)로 기록. 로컬 빌드에선 이미지 태그 끝자리에도 사용.
⚠
만료 검사는 UTC 자정 기준이며, 지난 경우 컨테이너 기동 자체가 실패합니다. (로컬 스크립트 기본값은 +1일이라 짧으니, 길게 쓰려면 명시 입력 권장)

PACKAGE — 허용 패키지 패턴

기본값 (빈값)자유 입력
설명이 Portable 빌드가 실링을 허용할 Android 패키지. 단일/와일드카드(com.foo.*)/콤마 다중 지원.
영향씰링 시 APK 패키지명을 path.Match로 이 패턴과 대조 — 불일치면 실링 거부. lc + license.json의 package에 기록.
⚠주의: 비워두면 snapshot 캡처 단계는 기본 com.doverunner.app으로 패키지별 엔드포인트를 호출합니다. 라이선스 패키지 검증을 의도하면 명시 입력 필요.

CUSTOMER_NAME — 고객명

기본값 (빈값)자유 입력
설명고객 식별/표시용 이름 (예: doverunner).
영향호스트측 license.json의 customerName에만 기록됩니다. 내장 lc나 Go 앱은 사용하지 않으며 강제되지 않는 메타데이터입니다.

ACCOUNT_ID — 계정 식별자

기본값 (빈값)자유 입력
설명라이선스에 묶이는 계정 식별자.
영향내장 lc의 account_id + license.json의 accountId에 기록. 런타임에 로깅되지만 강제되지는 않습니다.

REPORT — 리포트 환경

기본값 tst선택지 dev / tst / svc / custom / none드롭다운
설명리포트(탐지/크래시 등) 송신 대상 환경을 선택합니다.
동작 dev/tst/svc → 해당 환경 config의 api_base_url을 리포트 URL로 사용 · custom → REPORT_URL 값 사용 · none(또는 빈값) → 리포트 비활성.
영향build_portable.sh가 ^(dev|tst|svc|custom|none)$ 검증 후 → report_enabled 불리언(none/빈값=false, 그 외 true)과 report_url을 lc/license.json/config.yaml에 기록.

REPORT_URL — 커스텀 리포트 URL 조건부 필수

기본값 (빈값)자유 입력조건부 필수
설명리포트를 보낼 커스텀 URL.
영향REPORT=custom일 때만 사용/필수입니다. 그 외 값에선 무시됩니다.
⚠
REPORT=custom인데 REPORT_URL이 비어 있으면 build_portable.sh가 빌드를 실패시킵니다.

파이프라인 폼 밖: 리포지토리 변수

SecureOwl 다운로드(download_secure_owl.sh)에는 폼이 아니라 Bitbucket 리포지토리/Deployment 변수로 등록해야 하는 값들이 있습니다.

변수필수설명
SECURE_OWL_S3_BUCKET필수SecureOwl 아티팩트가 있는 S3 버킷
SECURE_OWL_S3_REGION선택리전 (기본 ap-northeast-1)
AWS 자격증명필수S3 접근용 (리포지토리 변수/OIDC 등)

예시 시나리오

1) 완전 오프라인(에어갭) 이미지 — 외부 연결 없이 실링

ENV          = svc          # svc 환경의 SecureOwl 사용
API          = none         # 오프라인: 내장 스냅샷 서버
VERSION      = 3.0.0.0
EXPIRE_DATE  =              # 비우면 +1년
PACKAGE      = com.acme.*
REPORT       = none         # 리포트 비활성

2) tst API에 붙는 휴대용 이미지 — 런타임에 외부 tst API 사용

ENV          = tst
API          = tst          # 런타임에 tst API 연결(오프라인 서버 미기동)
VERSION      = 3.0.0.0
PACKAGE      = com.acme.app
REPORT       = tst

3) 커스텀 리포트 URL

API          = svc
REPORT       = custom
REPORT_URL   = https://report.acme.internal/collect   # custom일 때 필수
PACKAGE      = com.acme.*
